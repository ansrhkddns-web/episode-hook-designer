# Reader Response Simulation

Use this reference to predict how readers may react to an episode, hook, or paid boundary. This is not a guarantee of market response. It is a practical stress test for comments, satisfaction, confusion, and drop-off risk.

## Simulation Targets

Predict:

- positive reaction
- complaint
- confusion
- comment target
- next-click motivation
- drop-off risk
- paid trust risk if relevant

## Reader Complaint Patterns

| Complaint | Usually Means | Fix |
|---|---|---|
| "그래서 언제 시작함?" | genre promise delayed | move premise proof earlier |
| "주인공 왜 저러고 있음?" | agency or motivation unclear | add visible goal, cost, or constraint |
| "또 당하네" | frustration exceeds reward | add counter-move, information gain, or small catharsis |
| "설명 너무 많다" | setup overtakes pressure | convert exposition into discovery under conflict |
| "낚시네" | payoff withheld unfairly | pay part of the promise before opening more |
| "갑자기 왜?" | emotional or causal step missing | add action evidence or transitional consequence |
| "댓글로 싸움 날 듯" | side-taking trigger works | preserve if intentional and not confusing |

## Comment Simulation

Generate comment directions, not fake exact metrics:

- theory comment: "X가 진짜 흑막인가?"
- punishment demand: "다음 화에서 꼭 응징해야 함"
- shipping: "이건 이미 감긴 거 아닌가"
- decision debate: "계약 받아야 한다 / 받으면 안 된다"
- status reaction: "드디어 알아보네"
- fear/worry: "저 선택 때문에 망할 것 같은데"

Use comments to check whether the episode gives readers a clear reaction object.

## Satisfaction Check

A reader can be unsatisfied in a useful way or a damaging way.

Useful dissatisfaction:

- wants the next payoff
- argues about choices
- worries about consequence
- theorizes from clues

Damaging dissatisfaction:

- feels promised payoff was dodged
- cannot understand why events matter
- feels protagonist has no agency
- sees only repeated frustration

If dissatisfaction is damaging, revise before strengthening the cliffhanger.

## Simulation Output

For commercial diagnosis, end with:

- likely comment direction
- likely drop-off line
- one fix that improves both satisfaction and click pressure
