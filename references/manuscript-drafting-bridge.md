# Manuscript Drafting Bridge

Use this reference when the user wants help writing an actual commercial webnovel episode from a hook plan. The goal is not to write decorative prose first. The goal is to convert the episode plan into scene-level drafting instructions that preserve retention, reward, and the next-click hook.

## Core Principle

원고는 회차 설계의 약점을 숨겨주지 않는다. 회차 기능, 보상, 압박, 엔딩 훅이 약하면 문장을 다듬어도 상품성이 올라가지 않는다. 먼저 장면 지시서를 만든 뒤 원고를 쓴다.

Use this structure before drafting:

```markdown

-

-

### 장면 순서

| 1 |  |  |  |  |  |  |
```

## Scene Unit Rule

Each scene unit should have:

- a pressure entering the scene
- a protagonist move
- an external response
- one information or emotion change
- a reason to move into the next scene

If a scene only explains information, convert it into:

- discovery during danger
- argument under time pressure
- test of a rule
- witness reaction
- public consequence
- forced choice

## Scene Count Guidance

For a typical serialized episode, use 4-7 scene units:

- 1 opening pressure
- 1-2 problem escalation scenes
- 1 protagonist move or discovery
- 1 reward or partial answer
- 1 consequence or hook scene

Use fewer scenes for high-tension episodes. Use more only when each scene changes pressure.

## Draftability Check

Before writing prose, verify:

- Does every scene have a function?
- Does the protagonist do something visible?
- Does the reader get a concrete reward before the ending?
- Is at least one reaction externalized through another character, system, crowd, object, rank, message, or result?
- Is the final scene a hook, not a cooldown?

If not, revise the packet.

## Prose Request Handling

If the user asks to write the episode, first produce or confirm the drafting packet unless the user already supplied one.

When writing prose from the packet:

- preserve scene function over decorative wording
- make the opening pressure visible quickly
- avoid front-loaded lore
- keep withheld information out of narration
- end at the hook moment, not after explaining the hook
- use dialogue only when it changes choice, power, relationship, or information

## Common Draft Failures

| Failure | Symptom | Fix |
|---|---|---|
| Summary draft | events are narrated after the fact | convert to live scene with pressure |
| Soft opening | characters talk before stakes appear | start with the problem already active |
| Hidden reward | reward exists only in thought | add witness, result, status, object, or reaction |
| Overexplained hook | final reveal is explained until tension cools | stop at reveal or forced response |
| No handoff | next episode has no immediate scene | write the exact next scene promise |
