## Example: Hunter Episode Hook

```markdown
| 항목 | 내용 |
|---|---|
| 회차 기능 | 첫 보상의 비용 노출 |
| 시작 압박 | 주인공이 숨긴 스킬 때문에 파티 생존률이 비정상적으로 오른다 |
| 주인공 행동 | 일부러 약한 척하며 보스의 패턴을 유도한다 |
| 독자 보상 | 히든 패턴을 읽고 아이템 드롭 조건을 알아낸다 |
| 체감 증거 | 시스템 메시지와 파티원의 반응이 동시에 뜬다 |
| 새 압박 | 드롭 아이템 소유권이 주인공에게만 귀속된다 |
| 엔딩 훅 | 길드 감시관의 단말에 주인공 이름이 붉게 표시된다 |
| 다음 화 질문 | 길드는 왜 주인공을 위험 등급으로 분류했나? |
| 댓글 포인트 | "저 길드가 뺏으러 오는 거 아닌가?" |
| 판정 | pass |
```

## Example: Romance/Rofan Episode Hook

```markdown
| 항목 | 내용 |
|---|---|
| 회차 기능 | 관계 거리 변화와 사회적 압박 |
| 시작 압박 | 여주가 계약 관계를 숨긴 채 무도회에 참석해야 한다 |
| 주인공 행동 | 상대의 체면을 지키기 위해 일부러 위험한 제안을 받아들인다 |
| 독자 보상 | 남주가 처음으로 공개적으로 여주 편을 든다 |
| 체감 증거 | 주변 귀족들의 호칭과 태도가 즉시 바뀐다 |
| 새 압박 | 둘의 관계를 아는 목격자가 왕비에게 보고한다 |
| 엔딩 훅 | 왕비의 초대장이 도착하고, 초대장에는 계약서 조항이 적혀 있다 |
| 다음 화 질문 | 왕비는 계약 관계를 어디까지 알고 있나? |
| 댓글 포인트 | "남주 이미 감겼다" / "왕비가 판 흔드는 듯" |
| 판정 | pass |
```

## Example: Weak Ending Repair

Weak:

```text
그날 밤, 주인공은 이상한 불안감을 느끼며 잠들었다.
```

Repair:

```text
잠들기 직전, 주인공의 휴대폰이 켜졌다.

[알림: 회귀 전 사망 예정자 명단이 갱신되었습니다.]

첫 번째 이름은 방금 자신을 도와준 사람의 이름이었다.
```

Why it works:

- vague mood becomes a concrete message
- the previous relationship becomes pressure
- the next-click question is specific
- readers can comment with prediction or worry
