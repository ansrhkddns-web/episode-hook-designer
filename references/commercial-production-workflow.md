# Commercial Production Workflow

Use this reference when the user wants a production-ready episode plan, not just ideas. The output should be clear enough for a writer to draft from and for an editor to approve or reject.

## Production Mindset

A commercial episode is not a summary. It is a working unit with:

- an opening pressure
- a protagonist action
- a visible reward
- a changed situation
- a next-click question
- a comment trigger
- a handoff to the next episode

If any item is missing, mark it as a revision point.

## Input Levels

Work with whatever the user provides:

- `concept only`: infer genre promise and produce a first-pass hook structure.
- `episode idea`: turn it into a production brief.
- `episode outline`: diagnose and reorder beats.
- `draft text`: identify hook failures and propose ending rewrites.
- `episode sequence`: build retention rhythm and paid-conversion pressure.

State assumptions briefly when input is thin.

## Production Steps

1. Define the episode's commercial job.
2. Identify the previous hook and decide how this episode responds.
3. Choose the reader reward.
4. Choose the protagonist action that earns or triggers the reward.
5. Externalize the reward through result, witness, reaction, status, object, or changed access.
6. Open new pressure that follows from the action.
7. Select a concrete ending object.
8. Write the next-click question.
9. Add one comment trigger.
10. Assign approval: pass, revise, or rebuild.

## Episode Job Menu

Use one dominant job:

- prove premise
- prove repeat reward
- deliver small catharsis
- reveal cost
- change relationship
- change public status
- answer a mystery
- expand the world
- force a choice
- trigger consequence
- set paid conversion
- defend paid purchase

## Editor Approval

Use clear production language:

- `pass`: ready to draft or serialize with minor prose polish.
- `revise`: usable episode but needs beat order, reward, or ending repair.
- `rebuild`: wrong episode function, missing reward, passive protagonist, or no click reason.

Do not say only "interesting" or "weak". Name the production failure:

- reward missing
- hook vague
- setup overweight
- protagonist passive
- genre reward mismatch
- paid promise unfair
- comment target absent
- ending after the peak

## Revision Priority

When fixing, use this order:

1. Move the genre promise earlier.
2. Give a visible reward.
3. Make the protagonist cause the turn.
4. Convert explanation into discovery under pressure.
5. Replace vague ending mood with a concrete hook object.
6. Add comment trigger from choice, unfairness, clue, or relationship tension.

Do not start by polishing prose if the episode function is wrong.
